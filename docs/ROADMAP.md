# ROADMAP

Este documento define a evolução planejada do aplicativo de hábitos pessoais, começando com ingestão de água e expandindo para um sistema completo de acompanhamento de rotina.

A lógica segue uma abordagem incremental: construir base sólida → adicionar inteligência → expandir para múltiplos hábitos.

---

## 🚀 Fase 1 — Fundamentos (MVP)

Objetivo: garantir que o app funcione perfeitamente para uso diário.

### Funcionalidades

- [x] Registro de ingestão de água
- [x] Definição de meta diária
- [x] Soma de consumo diário
- [x] Persistência de dados
- [ ] Interface mobile-first otimizada para iPhone
- [ ] Feedback visual ao registrar água

### Resultado esperado

Um app simples, rápido e confiável para uso diário.

---

## 📊 Fase 2 — Histórico e Estatísticas

Objetivo: dar visão clara de performance ao longo do tempo.

### Funcionalidades

- [ ] Dropdown de período:
  - Semana
  - Mês
  - Ano

- [ ] Histórico detalhado por dia:
  - Data
  - Total consumido
  - % da meta atingida

- [ ] Resumo estatístico:
  - Total do período
  - Média diária
  - Dias com registro
  - % de dias que bateram meta

- [ ] Cálculo de média (apenas dias com registro)

---

## 📈 Fase 3 — Engajamento e Consistência

Objetivo: aumentar uso diário e constância.

### Funcionalidades

- [ ] Streak (dias consecutivos batendo meta)
- [ ] Indicadores visuais de progresso
- [ ] Animação ao atingir meta diária
- [ ] Destaque de dias perfeitos (meta atingida)

---

## 🔔 Fase 4 — Notificações e Rotina

Objetivo: transformar o app em um assistente ativo.

### Funcionalidades

- [ ] Lembretes para beber água ao longo do dia
- [ ] Configuração de frequência de notificações
- [ ] Sugestões inteligentes baseadas no histórico

---

## 📱 Fase 5 — Expansão para Outros Hábitos

Objetivo: transformar o app em um sistema de hábitos completo.

### Novos módulos

- [ ] Corrida
- [ ] Sono
- [ ] Leitura
- [ ] Vitaminas
- [ ] Rotina diária

---

## 🧠 Fase 6 — Inteligência e Insights

Objetivo: transformar dados em decisões.

### Funcionalidades

- [ ] Insights automáticos (ex: "Você bebe mais água nos dias de corrida")
- [ ] Comparação entre semanas
- [ ] Tendências de comportamento
- [ ] Sugestões personalizadas

---

## 📌 Princípios de Prioridade

A ordem de desenvolvimento deve sempre seguir:

1. Funcionalidade básica funcionando
2. Experiência fluida
3. Consistência visual
4. Só então adicionar inteligência

---

## ⚠️ Regras de evolução

- Não adicionar múltiplas features complexas ao mesmo tempo.
- Sempre validar estabilidade antes de avançar de fase.
- Melhorar antes de expandir.
- Evitar overengineering no início.

---

## 🎯 Visão final

Um único app que centraliza hábitos essenciais do dia a dia, com baixa fricção e alta consistência de uso.
